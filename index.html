    История проекта

    Идея — создать простой и безопасный способ обмена файлами между членами семьи и друзьями, не требующий сложной настройки и публичных облачных хранилищ.

    • 2025, начало: Использовался Playit.gg для туннелирования, но возникли ограничения по трафику и времени сессии.
    • 2026, весна: Переход на Portmap.io + OpenVPN, но сложность настройки и необходимость ручного проброса портов.
    • 2026, июнь: Реализована архитектура на основе Flask (Python) и локального туннеля localtunnel, что дало HTTPS и простоту.
    • Текущая версия: Стек — Python + Flask, localtunnel, HTML-интерфейс в стиле Apple. Сервер запускается на домашнем ПК или Orange Pi.

    Версия 2.0 — добавлены тёмная тема, drag&drop, прогресс-бар, вкладки с документацией.

    OpenVPN и туннелирование

    VPN (Virtual Private Network) — это технология, создающая защищённое сетевое соединение через публичную сеть (Интернет). Трафик шифруется и передаётся через туннель, что обеспечивает конфиденциальность и обход ограничений.

    OpenVPN — одна из самых популярных реализаций VPN с открытым исходным кодом. Она использует SSL/TLS для шифрования и может работать как через UDP, так и через TCP.

    Как используется в нашем проекте:

    • Туннель до Portmap.io (ранее) — OpenVPN поднимал VPN-соединение до сервера-посредника, который пробрасывал порт на локальный Flask-сервер.
    • Текущее решение — мы отказались от OpenVPN в пользу localtunnel, который не требует сложной настройки и даёт готовый HTTPS-адрес.
    • Принцип туннелирования остаётся тем же: внешний запрос приходит на публичный адрес, перенаправляется по защищённому каналу к нашему локальному серверу.

    Для семейного использования шифрование не критично, но localtunnel обеспечивает базовый уровень безопасности через HTTPS.

    Команды для ручного запуска

    Все команды выполняются в командной строке или PowerShell с правами администратора.

    1. Запуск OpenVPN (если используется)
    openvpn --config AndreyVikodinov.FileSender.ovpn
    (путь к .ovpn-файлу должен быть указан или файл находиться в текущей папке)
    2. Запуск Flask-сервера
    python C:\Users\annaa\Desktop\iv\server.py
    (или перейдите в папку с сервером и выполните python server.py)
    3. Запуск localtunnel (с фиксированным поддоменом)
    lt --port 5000 --subdomain qwerty
    Если поддомен занят, замените qwerty на своё уникальное имя.
    4. Альтернатива — запуск всех сервисов одной командой (Python-скрипт)
    python launcher.py
    (предварительно создайте launcher.py с кодом из предыдущих ответов)
    Порядок запуска:
    1. Убедиться, что Flask-сервер запущен (команда 2).
    2. Запустить localtunnel (команда 3).
    3. Открыть HTML-интерфейс (локально или на GitHub Pages).
    4. В настройках указать адрес localtunnel (если он изменился).

    Требуется подтверждение

    Для продолжения работы необходимо подтвердить IP-адрес на странице localtunnel.

    Действия:
    1. Нажмите «Перейти на страницу подтверждения».
    2. Введите IP, который показан на открывшейся странице, и нажмите «Continue».
    3. Закройте вкладку и нажмите «Повторить запрос».

    Проверка действует 7 дней.
