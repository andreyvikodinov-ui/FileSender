    История проекта

    Идея — создать простой и безопасный способ обмена файлами между членами семьи и друзьями, не требующий сложной настройки.

    • 2025, начало: Использовался Playit.gg — ограничения по трафику.
    • 2026, весна: Переход на Portmap.io + OpenVPN, но сложность настройки.
    • 2026, июнь: Реализована архитектура на Flask и localtunnel.
    • Текущая версия: Стек — Python, Flask, localtunnel; интерфейс в стиле Apple.

    Туннелирование и localtunnel

    Туннелирование — технология, которая создаёт защищённый канал между вашим локальным сервером и интернетом. Запросы приходят на публичный адрес и перенаправляются к вашему компьютеру.

    Localtunnel — простой сервис, который выдаёт временный HTTPS-адрес и перенаправляет трафик на указанный локальный порт. Не требует сложной настройки, работает через WebSocket.

    В нашем проекте localtunnel связывает внешний адрес https://qwerty.loca.lt с вашим Flask-сервером на порту 5000. Все данные передаются по HTTPS.

    Команды для запуска

    Выполняйте в PowerShell или командной строке с правами администратора.

    1. Запуск Flask-сервера
    python C:\Users\annaa\Desktop\iv\server.py
    (или перейдите в папку и выполните python server.py)
    2. Запуск localtunnel (с фиксированным поддоменом)
    lt --port 5000 --subdomain qwerty
    Если поддомен занят, замените qwerty на своё уникальное имя.
    3. Альтернатива — единый запускатель
    python launcher.py
    (предварительно создайте launcher.py из предыдущих инструкций)
    Порядок действий:
    1. Запустите Flask-сервер.
    2. Запустите localtunnel.
    3. Откройте HTML-интерфейс (локально или на GitHub Pages).
    4. В настройках укажите адрес localtunnel, если он изменился.

    Требуется подтверждение

    Для продолжения работы необходимо подтвердить IP-адрес на странице localtunnel.

    Действия:
    1. Нажмите «Перейти».
    2. Введите IP, показанный на странице, и нажмите «Continue».
    3. Закройте вкладку и нажмите «Повторить».

    Проверка действует 7 дней.
